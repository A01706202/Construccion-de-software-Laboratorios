¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es una red global de redes y la www es una recopilación de información a la quese accede a través de internet.

¿Cuáles son las partes de un URL?

HTTPS: Transfiere información entre actores dentro de internet.
Subdominio: Lo que aparece entre el protocolo y el primer punto del dominio.
Dominio: se compone por nombre de dominio y la TLD, el primera es un nombre que identifica el nombre de la página web y la segunda da pistas sobre
el tipo de página que es.
Ruta: indica páginas y subpáginas del sitio, va después de la barra /
Parámetro: Indican diferentes cosas dependiendo de la página, ya sean búsquedas, publicidad, etc.
Etiqueta: Hacen scroll hasta un elemento en concreto, utilizan #

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita una representación de un recurso y solo recopila datos.
HEAD: Igual que el GET, pero sin el cuerpo de la respuesta.
POST: Envía una identidad a un recurso específico.
PUT: Remplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
PATCH: Aplica modificaciones parciales a un recurso.
DELETE: Borra un recurso específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Método post, porque los datos son incluidos al cuerpo de la petición en vez de obtener los datos adjuntos a la URL. Haciéndolo más seguro.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Método get, ya que es para pedir un recurso y por ende los datos se añaden a la URL.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que han sido removidos de HTML5, algunos son rev y charset (de los elementos a, link), longdesc (de img e iframe), nohref (area) y profile (head).

¿Cuáles son las diferencias principales entre HTML4 y HTML5?

- HTML5 define una sintaxis que son disponibles con HTML4.
- En HTML5 para definir el juego de caracteres, se introduce un nuevo atributo para "meta" (meta charset="UTF-8").
- HTML5 permite incluir elementos de SVG y MathML.
- Se simplifica el DOCTYPE (!DOCTYPE html).
- HTML5 se introducen elementos como footer, header, section, article, etc.
- HTML5 mejora las APIs.

¿Qué componentes de estructura y estilo tiene una tabla?

(table ... /table): Identifica y contiene todo lo relacionado a la tabla.
(caption ... /caption): Descripción de lo que la tabla es y de lo que contiene.
(thead ... /thead): Describe el tipo de información de cada columna.
(tbody ... /tbody): Información de la tabla.
(tfoot ... /tfoot): Describe la información del pie de la tabla.
(tr ... /tr): Contiene la información de una fila de la tabla.
(th ... /th): Contiene la información en un encabezamiento.
(td ... /td): Contiene la información en una casilla.
(colgroup ... /colgroup): Agrupa columnas para fines de formateo.
(col /): Define una sola columna de información dentro de la tabla.

¿Cuáles son los principales controles de una forma HTML5?

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?
1) Fase de planificación: Se prepara el diseño y luego implementación del sistema. Se necesita definir el alcance del proyecto,
justificarlo y escoger una metodología para el desarrollo.
2) Fase de análisis: Se busca concretar una serie de requisitos que son los que regirán el nuevo sistema.
3) Fase de diseño: Se determina como el nuevo sistema de información cumplirá con los requisitos aplicables. Identifica soluciones,
las evalua y se elige la más conveniente.
4) Fase de desarrollo: Donde se aplican los cambios al sistema.
5) Fase de integración y periodo de pruebas: Corroborar que el diseño propuesto cumpla con los requisitos establecidos.
6) Fase de implementación: Procede a la instalación del software y hardware elegidos, someter pruebas, crear documentación pertinente
y capacitar usuarios.
7) Fase de mantenimiento: Consiste en introducir los ajustes necesario del día a día.

¿Cuál es el ciclo de desarrollo de sistemas de información?
1) Planificación del concepto: Se define los sistemas a diseñar y determinan el alcanse del proceso.
2) Definir los requisitos: Se definen los requisitos que se tienen que cumplir por el proyecto.
3) El diseño: Se crea un diseño técnico con el que previsualizan el trato que se les dará a los requisitos.
4) Fase de desarrollo y pruebas: Crean el software y la arquitectura física necesaria para albergar la base de datos del sistema. Cuando
se termina, se realizan pruebas para ver el funcionamiento y si es necesario, hacer correcciones.
5) Puesta en marcha: El sistema es pasado a los usuarios finales para que se empiece a utilizar.
6) Operaciones y mantenimientos: Se controla el sistema para asegurar que cumplan los requisitos y se le da mantenimiento para que funcione.
7) Disposición: Comprende el fin del ciclo de vida del sistema y su retiro, se siguen pasos sistemáticos para finalizarño en un entorno de
seguridad que permita conservar la información útil o sensible.

    Preguntas laboratorio CSS

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Que se utilice cuando se quiera definir reglas de estilos en navegadores antiguos y para hacer que unas propiedades tomen mayor importancia que otras sobreescritas.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Porque puede afectar a la optimización de la página web y puede que el tamaño de la imagen no concuerde con el de la página, haciendo que se viera mal.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

% se utilizaría para establecer proporciones adecuadas en relación a la pantalla, el px para que los elementos web se pueda visualizar con tamaños precisos y el pt para valores absolutos, pero por esta razón no es muy recomendable porque es difícil que se adaptenal tamaño de pantalla de cada usuario.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Porque de esta manera ya no consumen tanto espacio, tiempo y recursos de los navegadores web para procesarlos.
